WebPDA

Access your real-time data in real-time, from the web

Download this project as a tar.gz file Download this project as a .zip file
WebSocket based Process Data Access

WebPDA is a protocol for accessing real-time data using WebSocket.

Real-time data is usually related to a process, so it is also called process data. For example, current temperature of a furnace, price of a stock, blood pressure of a person, CPU and memory usage of a server... These variables are called Process Variable (PV) in WebPDA.

Internet of Things, SCADA, finance, health, weather, environment monitoring, measurement and control systems, data center, whenever you need to access your real-time data from the web, WebPDA can ease your life.

Features

  • Real-time server side push.
  • Highly efficient: use binary format for values and JSON for other info.
  • Extensible to arbitrary data sources. EPICS support is included.
  • Protocol specification is language independent. Currently, it provides server side implementation on JSR 356 and client side implementation on JavaScript.
  • Focus on your data, instead of communications. Super easy client side API.

JavaScript API

Few lines code would allow you to connect to your data 

var wsUrl = "ws://localhost:8080/webpda/webpda";
var wp = new WebPDA(wsUrl, "myname", "mypassword");
var pv = wp.createPV("sim://noise", 1000, false);
pv.addCallback(function(evt, pv, data) {
    switch (evt) {
    case "conn": //connection state changed
        break;
    case "val": //Value changed. 	
        break;
    case "bufVal": //Buffered values changed        
        break;
    case "error": //error happened        
        break;
    case "writePermission": // write permission changed.        
        break;
    case "writeFinished": // write operation finished. 
        break;
    default:
        break;
    }
});

//Write PV.
pv.setValue(10);

//Close PV
pv.close();

Pre-wrapped widgets allows binding a widget with PV in one line of code 

					<div class="webpda-widgets" data-widget="rgraph-gauge"
					data-pvname="sim://noise"></div>
